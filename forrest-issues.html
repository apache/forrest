These are the open issues for our upcoming release listed in our issue tracking system (see all open issues). The listing below is regenerated on each Forrest run.

need to silence the build messages from JCS store

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-214

There are too many messages coming from the JCS store when 'forrest' is run. This would confuse users. Cocoon added some configuration recently.

Custom sitemap.xmap causes hang

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-202

If there is a sitemap.xmap file in the "project.content-dir", the build process of a site (target: "site") will hang with an OutOfMemoryError.

local stylesheets for a user project not found

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-198

When a user project defines their own sitemap and stylesheets, then the sitemaps are over-ridden okay, but their local stylesheets are not found.

Use code-names for skins

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-181

We should be using codenames for the skins, so we do not tie them to a site that may change skin over time.

Have @class attribute in DTD for every tag

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-174

Add @class attribute to all tags so that the extra-css section in the skinconf can be put to good use.

Finish off copyless behaviour

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-172

The "true" copyless behaviour is not quite fully implemented.

build test - target fails, still refers to old dist/shbat/ stuff

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-171

Runing the 'build test' fails. It has not yet been adjusted after the merge of the "copyless" branch.

InvocationTargetException on shutdown from forrest run

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-170

After using 'forrest run' then Ctrl-C to shutdown the demo webapp,
an InvocationTargetException is issued.

linkrewriting not done for aggregate pages

http://issues.cocoondev.org/jira//ViewIssue.jspa?key=FOR-147

site: and ext: links are not rewritten for site.html or site.pdf
