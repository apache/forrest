[FOR-281] Upgrade the Using Forrest document to reflect new project sitemap method

[FOR-279] use of Ant xmlproperty on the skinconf.xml causes trip to website to get DTD

http://issues.cocoondev.org//browse/FOR-279

Doing 'forrest war', the build system uses the <xmlproperty> Ant task to load the skinconf.xml which causes Ant to try to get the DTD from the website. The Ant xmlcatalog cannot yet be used with Ant xmlproperty.
Our Dave has submitted patch at http://issues.apache.org/bugzilla/show_bug.cgi?id=27053 (not complete and needs investigation).

[FOR-277] investiagte use of forrest-0.6-dev as webapp with Tomcat 5

http://issues.cocoondev.org//browse/FOR-277

Some people report having troubles running Forrest as a webapp under Tomcat. Needs investigation before 0.6 release.

[FOR-255] catalog entity-resolver fails and gets skinconf DTD from website when using document()

http://issues.cocoondev.org//browse/FOR-255

The Catalog Entity Resolver fails and so follows the systemIdentifier to get the website copy of the DTD. This seems to be happening due to the use of the XSLT document() function.
