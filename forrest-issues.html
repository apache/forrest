[FOR-255] catalog entity-resolver fails and gets skinconf DTD from website when using document()

http://issues.cocoondev.org//browse/FOR-255

The Catalog Entity Resolver fails and so follows the systemIdentifier to get the website copy of the DTD. This seems to be happening due to the use of the XSLT document() function.

[FOR-254] validate target

http://issues.cocoondev.org//browse/FOR-254

the validate target should be run by default

[FOR-251] start script for UNIX bin/forrest should not be bash-specific

http://issues.cocoondev.org//browse/FOR-251

There are some commands in the bin/forrest script that are specific to the bash shell. On most systems the default /bin/sh is actually bash. However, not so on one significant OS. We should be neutral.

As a workaround the script has been set to be /bin/bash

[FOR-247] Tigris *.css need cleanup to remove our local changes

http://issues.cocoondev.org//browse/FOR-247

We have made some local changes to our copies of the tigris *.css which actually need to go into an over-riding css. Also remove our license notice which was added by accident.
http://marc.theaimsgroup.com/?l=forrest-dev&m=109169132601758

[FOR-245] changing skins and going out in the public view

http://issues.cocoondev.org//browse/FOR-245

It almost time to change the skin on our public Forrest website.

[FOR-224] All documentation needs review, to be up-to-date with copyless procedure

http://issues.cocoondev.org//browse/FOR-224

Since the change to copyless there is stacks of documentation that needs to be modified.

[FOR-223] build target "release-dist" still refers to old dist/shbat/ stuff

http://issues.cocoondev.org//browse/FOR-223

The "release-dist" target needs to be fixed and tested, since the changes for "copyless".

[FOR-202] Custom sitemap.xmap causes hang

http://issues.cocoondev.org//browse/FOR-202

If there is a sitemap.xmap file in the "project.content-dir", the build process of a site (target: "site") will hang with an OutOfMemoryError.

[FOR-198] local stylesheets for a user project not found

http://issues.cocoondev.org//browse/FOR-198

When a user project defines their own sitemap and stylesheets, then the sitemaps are over-ridden okay, but their local stylesheets are not found.

[FOR-172] Finish off copyless behaviour

http://issues.cocoondev.org//browse/FOR-172

The "true" copyless behaviour is not quite fully implemented.
