Version 0.6.1 (not released yet)

Changes to Build

  • Change Ant build property name "version" to "forrest.version" to prevent collisions with other build systems when using the Ant "import" task. (DC)

Version 0.6 (2004-10-16)

Changes to Code Base

  • Eclipse plugin code added to scratchpad (donated by Burrokeet). Initial work, needs enhancement. (RDG)
  • Locationmap modules added. Not in use yet. (NKB) Thanks to Unico Hommes.
  • Forrestdoc code added to scratchpad; previously was Jakarta Alexandria (NKB)
  • PDF pages receive skinconf properly (through the skinconf pipeline). (DB) Fixes FOR-138.
  • FAQ page does not need to provide it's own extra index. (DB) Fixes FOR-154.
  • Brokenlinks file is now generated in project.home rather than forrest.home (JJP) Fixes FOR-165.
  • Add missing xdocs from the validate.exclude property. Allowing to exclude files. (JJP) Fixes FOR-130.
  • Improved resolving image references in PDFs. (DB) Fixes FOR-114.
  • Lucene-index target works. If search@provider="lucene" is set create a search index. Build site and run targets as per project directory instead of forrest.home. (JJP)
  • Clean up first Lucene search implementation in Forrest. (JJP)
  • Projects can use a local CatalogManager.properties to declare additional Catalogs for DTDs and other entities. (DC)
  • Improve look of PDF pages. (DB)
  • Fix relative links and image references in aggregate pages. (DB) Fixes FOR-148.
  • Worked search functionality into all skins. Fixed query parameters for Google site search. Lateral search box was incompatible with new skinconf.xml format. (JJP) Thanks to Florian G. Haas. Fixes FOR-9.
  • Upgrade to cocoon 2.1.5 (JJP)
  • Removed author and cvs-id tags from code and xdocs. (DB) Fixes FOR-123.
  • Added support for docbook version 4.1.2. Allows importing files from various editors (Open office, Abiword). (JJP)

Changes to Documentation

  • Added new document to facilitate upgrading to v0.6 (DC)
  • Added a How-To: Generate an ASF mirrors page using interactive web form using ihtml to include additional html form elements. (DC)
  • Added a How-To: How to create a PDF document for each tab to create subject-specific aggregated documents. (TS) Thanks to Stephan E. Schlierf. Fixes FOR-218.
  • Created the How-To section of the documentation. Continue Dianna Shannon's good work with designing the DTDs, concepts, and documentation effort at Cocoon. (DC)
  • Added a mechanism to enable impoverished XML tools that do not use the Catalog Entity Resolver to fetch the DTDs direct from the website. See upgrading to v0.6 Please help to upgrade our /dtd/.htaccess file (see issue FOR-107) and please help to address the SVN Issue 1233. (DC) Fixes FOR-107.
  • Added new documentation about Searching Forrest-built documentation which describes both Lucene and Google search. (JJP) Thanks to Florian G. Haas. Fixes FOR-9.
  • Began documentation for new forrestbot; moved stable forrestbot docs (DB)
  • Replace URLs in all docs to be http://forrest.apache.org/ (JJP)
  • Forrest is now a top-level Apache project. The new website is at http://forrest.apache.org/ (DC)

Changes to Build

  • New 'clean-dist' target for build.xml, which is now the default target. (DB) Fixes FOR-108.
  • The 'validate-site' build target will validate the intermediate xdocs. (JJP)
  • Support running Forrest on Win9x. (DB) Thanks to Pierrick Brihaye. Fixes FOR-113.
  • Don't copy unneeded project files in 'war' and 'webapp' targets. (DB) Fixes FOR-207.
  • Add more desriptions to forrest targets. Highlight the most common ones. (DB) Fixes FOR-216.

Version 0.5.1 (2003-10-06)

Changes to Build

  • forrest.antproxy.xml script now properly fails when Cocoon fails (JT)

Version 0.5 (2003-09-13)

Changes to Documentation

  • Added new document to facilitate upgrading to v0.5 (JT)

Version 0.4 (2003-02-12)

Changes to Documentation

  • Added page for ForrestBar on the site with a simple link and an autoinstall link to the forrestbar.xpi generated by Gump every day. (NKB)

Version 0.3 (2003-01-30)

Changes to Code Base

  • Update to 2002-12-26 Cocoon. (JT)
  • Updated latest Cocoon jars: chain metamodule name set to "forrest", and it is used instead of the default one. (NKB)

Changes to Documentation

  • Added enhancements to status.xml to comply with the needs for a complete status file, like other "STATUS" text files at Apache. Stylesheets not yet done. (NKB)

Changes to Build

  • Allow configuration of a project local-catalog and the entity resolver verbosity level, via forrest.properties configuration. This replaces tokens in the default cocoon.xconf (DC)

Version 0.2 (2002-11-16)

Version 0.2rc1 (2002-11-13)

Changes to Code Base

  • User projects can now override the default cocoon.xconf and logkit.xconf files, by placing these files in src/documentation/conf (JT)
  • added NekoDTD-based DTD documentation using a custom Cocoon Generator (SN)
  • draft forrest toolbar for Mozilla (NKB)
  • Initial import of Libre facility - replacement for */book.xml (SN) Thanks to Marc Portier.
  • Refactored site generation stuff to use skins; still need to integrate with xforrest descriptor. Added jakarta-site, xml-apache-site, cocoon-printer skins. (NKB)
  • Fixed a bug where the log files weren't being created. Thanks to Vladimir Bossicard for reporting. (JT)

Changes to Documentation

  • Added new document Standards Compliance Thanks to Robert Koberg. (DC)
  • Allow user to specify what XML files to validate via forrest.validate.{includes,excludes} properties. (JT)
  • Added new document Your Project - getting started with using Forrest (DC)
  • Added to the "forrest-site" skin the ability to split source code larger than 80 chars in two or more lines. (NKB)
  • Replace the "forrest-site" skin with the new "bert" skin. (DC) Thanks to Bert Van Kets.
  • Added breadcrumb trail script from the Avalon site skin. (NKB) Thanks to Leo Simons.
  • Added initial "Dream List" by harvesting statements from the forrest-dev mail list. (DC)
  • Added preliminary version of a scarab-like skin. (NKB)
  • Fixed links in Forrest home page and added: contributing, mailing list, list archives, who we are. (NKB)
  • Fixed all broken links in the Forrest site. (JT)
  • Simplified the sitemap-stylesheet contract. Instead of 'dir' and 'resource' parameters with varying meaning, there is just one 'path' parameter. (JT)
  • Fixed broken PDFs in most subdirectories. (JT)
  • PDFs for faq, changes and todo now work. (JT)
  • Made forrest-site skin fully generic, usable with any project. Skins are configured through an external 'skinconf.xml' file, specified by the project. (JT)
  • Fixed bug where multiple copies of the same image where generated, one for each directory. Now all image links are page-relative, leading to a single copy. (JT)
  • Fixed tab links, which were previously hardcoded and forrest-specific. They are now page-relative. tabs.xml file entries no longer have to end in '/'. (JT)
  • Ensure that initial DTDs are valid and that the initial XML instances can be validated using these DTDs and OASIS Catalogs. (DC) Thanks to Steven Noels.

Changes to Project Administration

  • Initial Import (SM)

Changes to Design

  • Designed a new skin for the Forrest documents. Particular attention to cross-browser issues. (BK)

Changes to Build

  • Validate project sitemaps and choice of skin (JT)
  • User projects can now add custom classes and jars to Forrest doc builds. Useful for custom sitemap components, or overriding Forrest-provided jars. (JT)
  • Added the forrestbot - infrastructure and configuration for remote project building. (SN) Thanks to Marc Portier.
  • Added a draft DTD for the Gump Descriptor forrest.xgump (SN)
  • Forrest website is being automatically built and published to krysalis.org (NKB)
  • Added simple xsl stylesheet for forrest.xgump. In IE6 and Mozilla 6 it renders using this, in older ones it uses the more simplifies CSS version. (NKB)
  • Added Forrest to Gump runs. (NKB)
  • Added simple css stylesheet for forrest.xgump. In IE and Mozilla 6 it renders. (NKB)
  • Merged forrest needed info in forrest.xgump. (NKB)
  • Imported Centipede as a build system. (SN)
  • Building Forrest no longer requires a network connection (for downloading Centipede). It is buildable by typing 'ant' or with the provided build scripts. (JT)
  • "Change History" images (update.jpg, add.jpg etc) are now provided by default by Forrest (in images/), and need not be supplied by user projects. (JT)
  • The locations where Forrest expects doc files to be is now configurable via properties, that can be set in forrest.properties. (JT)
  • User projects may now specify their own sitemap, overriding the Forrest-provided one. (JT)
  • Updated Cocoon to 2.1-dev to use xsltc and new CLI status messages. Logs are in the build/work/WEB-INF/logs directory and the list of broken links is in the build directory. (NKB)
  • Updated Centipede stuff to latest changes: - added lib/endorsed dir for endorsed jars that are put in global calsspath - changed the build scripts to not make direct reference to jar files - added "examples" dir in src and creation of examples jar in buils - added "testmodule" target to be run before committing stuff - added "gump" build to be run by Gump (NKB)
  • Updated project descriptor: now it should be usable by Gump. Added basic Forrest descriptor. (NKB)
