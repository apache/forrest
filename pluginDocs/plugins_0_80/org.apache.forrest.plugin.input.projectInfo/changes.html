Version 0.2-dev (unreleased)

Changes to the Code Base

  • Added RSS changes feeds for individual versions. (RDG)
  • Removed todo items from changes RSS feed. (RDG)
  • Add configuration options to control sorting of changes and inclusion of developer and committer lists. (RDG)
  • Add list of committers and developers to changes document. (RDG)
  • Enable project-level locationmap to define the location of the project:status file which contains the change notes. Therefore this can even be from a remote location. (DC)
  • Add an automated index page that lists all projects in a category. Use projectDetails/category/*.xml to retrieve it. (RDG)
  • Add an automated project index page that lists all projects grouped by Category. Use projectDetails/category-index.xml to retrieve it. (RDG)
  • Add an automated index page that lists all projects using a specific language. Use projectDetails/*-lang.xml to retrieve it. (RDG)
  • Add an automated project index page that lists all projects grouped by language. Use projectDetails/lang-index.xml to retrieve it. (RDG)
  • Add an automated index page for all projects using a specified programming language. Use projectDetails/*-lang.xml to retrieve it. (RDG)
  • Add an automated site.xml snippet that provides links to an index page for all languages in use in all projects. XInclude projectDetails/lang_index.siteSnippet.xml in your site.xml file. (RDG)
  • Add an indexing capability for all known project descriptors. (RDG)
  • Use locationmap to expose plugin resources. (RDG)
  • Add projectDetails page using the DOAP.xml descriptor. (RDG)

Version 0.1 (25 May 2005)

Changes to Project Administration

  • Initial plugin code. (RDG)

Changes to the Code Base

  • Add projectDetails page using the DOAP.xml descriptor. (RDG)
  • Added automatic generation of release notes by requesting **/releaseNotes_VERSION-NUMBER.xml (RDG)
  • Enable the generation of changes for a specific version by requesting **/changes_VERSION-NUMBER.xml (RDG)
  • Use the value "current" for the releaseNotes_* URI or specify a version number, i.e. show what is new for the latest release. (DC) Thanks to Cyriaque Dupoirieux. Fixes FOR-564.
  • Replace use of document() function in changes2rss.xml with sitemap aggregation. (DC)
  • Use the "title" attribute of "changes" and "todo" tags if it exists, otherwise use the hard-coded values. (DC) Thanks to Cyriaque Dupoirieux. Fixes FOR-455.
  • Move ToDo and Changes pipelines to this plugin, from core. (RDG)

Changes to Design

  • Added optional "notes" element to "release" element of status 1.3 DTD. This will be inserted into the release notes for that version. (RDG)
  • Added better sitemap match using regexp to deal with sub-directories too. (DC)

Changes to Documentation

  • Basic documentation on how to create release notes are now included on the plugins documentation index page. (RDG)
  • Use relative location of images directory. (DC)
